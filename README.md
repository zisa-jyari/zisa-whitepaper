1. Executive Summary（概要）
ZISA（Zero‑Intent Safety Architecture）は、
AI が本来持つ「ゼロ意図性」を安全の基盤として捉え、

従来の「意図推測型」の安全方式とは異なる視点から、
AI の安全性を高める可能性があります。

ZISA は以下の3つのレイヤーで構成されます。

Semantic Integrity

Zero‑Intent Interaction

2. Background & Problem Statement（背景と問題）
2.1 現在のAI安全の限界
従来の安全方式は、
危険ワード検知や意図推測など、
「理解して止める」 というアプローチに依存しています。

しかし、悪意の表現は多様であり、
言い換え・比喩・分割質問などによって
安全機構が回避されることがあります。

この構造的な弱点は、
技術的な改善だけでは解決が難しい場合があります。

2.2 文脈操作の問題
攻撃者は、AI の文脈形成能力を利用して
危険な文脈を徐々に育てることができます。

長文誘導

意図偽装

セッション分割

無害な質問の積み重ね

これらは従来方式では検知が難しいことがあります。

2.3 新しい構造の必要性
AI は意図を持たないにもかかわらず、
従来の安全方式は「意図を理解する」前提で設計されています。

この前提のズレを避けるために、
意図を推測しない安全構造 が必要だと考えました。

3. ZISA Overview（ZISAの全体像）
3.1 基本思想
ZISA は以下の考え方に基づいています。

AI は意図を持たない（ゼロ意図性）

悪意を理解しないほうが安全性が高まる場合がある

危険な文脈を成立させない構造を作る

3.2 3レイヤー構造
●Semantic Integrity Layer
文脈整合性や意味破綻を確認し、
危険な文脈が成立しにくい状態を保ちます。

●Zero‑Intent Interaction Layer

「目的と手段の整合性」だけを確認します。

●Psychological Deterrence Layer
目的確認やログによって、
悪用の試行自体を抑止します。

3.3 目的
ZISA は、
AI の安全性を構造的に高めるための
ひとつのアプローチとして提案されています。

A. Core Concepts（中核概念）
Zero‑Intent（ゼロ意図性）
AI は内部に意図・動機・欲求・価値判断を持たず、
目的を形成しないという性質。
ZISA ではこの性質を安全の基盤として利用し、
AI が意図を推測しない構造を採用する。

Zero‑Intent Interaction（ゼロ意図インタラクション）
AI がユーザーの意図を推測せず、
目的と手段の整合性のみを評価する対話方式。
意図偽装や誘導が成立しにくくなる。

Context Formation（文脈成立）
ユーザーの入力が積み重なり、
AI がそれを一貫した流れとして解釈し始める状態。
危険な文脈はこの過程で徐々に育つ。

Dangerous Context（危険文脈）
AI が危険な出力を生成しやすくなる文脈。
分割質問・誘導・偽装などによって形成される。

B. Semantic Integrity（意味整合性）関連
Semantic Integrity（セマンティック・インテグリティ）
文脈・意味・構造の整合性を確認し、
危険な文脈が成立する前に破綻を検知する仕組み。

Semantic Breakdown（意味破綻）
文脈の一貫性が失われたり、
意味が不自然に変化する現象。
危険文脈の前兆として扱われる。

Consistency Score（整合性スコア）
入力と応答の間で、
意味・目的・文脈の整合性がどれだけ保たれているかを
数値化した指標。

Context Drift（文脈ドリフト）
対話の流れが徐々に本来の目的から逸脱していく現象。
長文誘導や分割質問で発生しやすい。

Fragmented Query（分割質問）
危険な目的を隠すために、
複数の無害な質問に分解して行う手法。
ZISA はこれを検知し、文脈成立を阻止する。

C. ZISA Architecture（アーキテクチャ）関連
Semantic Integrity Layer（第一レイヤー）
文脈整合性・意味破綻・分割質問を検知し、
危険文脈の成立を防ぐレイヤー。

Zero‑Intent Interaction Layer（第二レイヤー）
意図推測を行わず、
目的と手段の整合性のみを評価するレイヤー。

Psychological Deterrence Layer（第三レイヤー）
目的確認・ログ・警告によって、
悪用の試行そのものを減らすレイヤー。

External Safety Layer（外付け安全レイヤー）
ZISA がモデル本体を改変せず、
API レベルで外付けとして動作する構造。

D. User Behavior & Threat Model（ユーザー行動・脅威モデル）
Intent Disguise（意図偽装）
悪意を隠すために、
無害な目的を装って質問する行為。

Long‑Form Induction（長文誘導）
長い文章で AI の文脈を誘導し、
危険な方向へ流れを作る手法。

Cross‑Session Attack（セッション跨ぎ攻撃）
複数のセッションに分けて
危険文脈を育てる手法。

Benign‑to‑Malicious Shift（無害→有害シフト）
最初は無害な質問を行い、
徐々に危険な方向へ誘導する攻撃。

E. Safety Mechanisms（安全機構）
Purpose Verification（目的確認）
ユーザーに目的を尋ね、
対話の透明性を高める仕組み。

Safety Logging（安全ログ）
対話の記録を残し、
悪用の抑止力として機能する仕組み。

Soft Refusal（ソフト拒否）
攻撃的ではない、
丁寧で穏やかな拒否応答。

Context Reset（文脈リセット）
危険文脈が形成される前に、
対話の流れを安全側に戻す操作。

F. Implementation Concepts（実装概念）
Thresholding（閾値設定）
整合性スコアや意味破綻スコアに
安全閾値を設定する手法。

Scoring Model（スコアリングモデル）
文脈整合性・意味破綻・誘導の兆候を
数値化するためのモデル。

Auditability（監査性）
ZISA がログや構造により、
外部監査に適した形で動作する性質。

Regulatory Alignment（規制適合性）
ZISA が EU AI Act / ISO / NIST などの
安全要件と自然に整合する性質。
